## 🎬 第一部分：项目总体框架介绍（3-4分钟）

### 🏗️ 技术架构概述

#### 开场白
```
老师好，今天我要为您介绍的是一个基于知识图谱的医疗信息管理系统。
这个项目采用前后端分离的架构设计，旨在将复杂的医疗知识关系通过
图谱形式直观展现，并结合个人健康管理功能。
```

#### 系统架构层次
1. **数据存储层**
   - Neo4j图数据库
   - 存储医疗实体和关系
   - 支持复杂图查询

2. **前端展示层**
   - Vue 3 + Composition API
   - Tailwind CSS响应式设计
   - SVG动态图形渲染

3. **功能模块层**
   - 知识图谱可视化模块
   - 智能问答交互模块
   - 个人健康管理模块

#### 讲解要点
```
我们的系统分为三个主要层次：

1. 底层数据存储层：
   使用Neo4j图数据库存储医疗知识图谱，包括药物、疾病、基因、
   靶点、通路等五大类医疗实体及其复杂关系网络。Neo4j的图结构
   天然适合存储这种多对多的复杂关系，支持高效的图遍历查询。

2. 前端可视化层：
   采用Vue 3框架结合Composition API进行响应式开发，通过SVG
   技术实现高性能的图谱可视化渲染。支持节点拖拽、缩放、高亮
   等丰富的交互操作，用户可以直观地探索医疗知识之间的关联。

3. 智能应用层：
   在图谱可视化基础上，我们构建了两大核心应用模块：
   
   a) 智能问答系统：基于图谱数据实现自然语言查询，用户可以
      通过对话方式获取专业医疗知识，如"阿司匹林可以治疗哪些
      疾病？"系统会从图谱中检索相关关系并生成准确回答。
   
   b) 个人健康管理：将专业医疗知识与个人健康数据结合，提供
      症状记录、用药管理、就医提醒等功能，让图谱中的专业知识
      能够指导用户的日常健康决策。

这种分层架构设计既保证了数据的专业性和准确性，又实现了
知识的可视化展现和智能化应用。
```


### 🔍 数据库查询演示

#### 实际操作展示
1. **简单查询示例**
   ```cypher
   MATCH (d:Drug)-[r:TREATS]->(dis:Disease)
   WHERE d.name = "阿司匹林"
   RETURN d, r, dis
   ```

2. **复杂关系查询**
   ```cypher
   MATCH path = (d:Drug)-[:TARGETS]->(t:Target)-[:ASSOCIATED_WITH]->(g:Gene)
   WHERE d.name = "阿司匹林"
   RETURN path LIMIT 10
   ```

#### 讲解要点
```
通过Cypher查询语言，我们可以灵活地检索图谱中的复杂关系。
比如这里查询阿司匹林相关的所有治疗关系，可以看到它与多种
心血管疾病存在治疗关系，这些数据为我们的前端可视化和智能
问答提供了丰富的内容基础。
```
